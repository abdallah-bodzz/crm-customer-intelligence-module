Core tables — build in this order
dim_customer — SCD Type 2
Business key: customer_unique_id. Dedup logic: group customer_id entries by customer_unique_id, keep latest shipping address. Columns: customer_unique_id, customer_id_latest, zip_code, city, state, valid_from, valid_to, is_current, geo_lat, geo_lng (joined from geolocation via zip centroid).
warehouse.dim_customer
dim_date — full calendar
Generate Sept 2016 – Dec 2018. Columns: date_key (INT YYYYMMDD), full_date, year, quarter, month, week, day_of_week, is_weekend, is_brazilian_holiday (manually populate public holidays), fiscal_month. Build once in SQL via recursive CTE or tally table.
warehouse.dim_date
fact_orders
One row per order. Key derived cols: delivery_delta_days (actual – estimated), is_late (bit), days_to_approve, days_to_carrier, order_status_clean. Join to dim_customer on customer_unique_id, dim_date on order purchase date. Include payment_total from payments rollup.
warehouse.fact_orders
fact_order_items + dim_product + dim_seller

Phase 5 Python ML pipeline Days 7–11 · 10–14 hrs
Model 1 — sentiment (run first, feeds other models)
sentiment.py — VADER on review text
Pull review text from warehouse.dim_review. Run VADER SentimentIntensityAnalyzer on review_comment_message. Write compound_score + sentiment_label back to mart.sentiment_scores. Then aggregate per customer: avg_compound, sentiment_trend (last review score – first review score, positive = improving). Write avg_sentiment to mart.clv_features and mart.customer_360.
vaderSentiment NLP
Model 2 — RFM segmentation
segmentation.py — RFM + K-means
Pull mart.rfm_features. Scale R/F/M with StandardScaler. Elbow + silhouette for K (expect K=4–6). Label clusters with business names (Champions / Loyal / At Risk / Hibernating / Lost / New). Write km_cluster + rfm_segment + rfm_label to mart.rfm_features. Save model as /models/kmeans_rfm.pkl.
scikit-learn unsupervised
Model 3 — CLV regression
clv_model.py — XGBoost regressor
Target: actual_gmv_post_cutoff (computed from fact_orders after Sept 2018). Features: all of mart.clv_features + avg_sentiment. Train/test temporal split. Tune with cross-val. Output: clv_predicted_6m + clv_ci_lower/upper (use quantile regression or bootstrap CIs). Write back to mart.clv_features. Save model as /models/xgb_clv.pkl. Log RMSE + MAE to reports/model_metrics.md.
xgboost regression
Model 4 — churn classifier
churn_model.py — XGBoost + threshold tuning
Churn definition: no order in 180 days from last_order_date (pre-cutoff customers only). Features: recency_days, order_frequency, avg_delivery_delta, avg_review_score, avg_sentiment, tenure_months, pct_late, state (encoded). Class imbalance: use scale_pos_weight. Tune threshold at 0.3 (not 0.5) — optimise for recall. Write churn_probability + is_churn_risk to mart.customer_360. Log precision/recall/F1/AUC. Save /models/xgb_churn.pkl.
xgboost classifier
Model 5 — next-purchase timing
next_purchase.py — Weibull AFT (lifelines)
Only run on customers with 2+ orders (inter-purchase intervals exist). Fit WeibullAFTFitter on duration (days between orders) + covariates (state, category, AOV bucket). Output: expected_next_purchase_days + survival probability at 30/60/90 days. Write to mart.customer_360. This is the differentiation model — document it prominently.
lifelines survival analysis
Post-ML — action queue population
Populate mart.crm_action_queue
Rule engine (Python, simple if-else on model outputs): churn_prob >= 0.6 AND clv_predicted > median → RETENTION_CAMPAIGN HIGH. churn_prob >= 0.6 AND clv_predicted <= median → REACTIVATION MED. rfm_segment = Champions → VIP_UPGRADE MED. All others → MONITOR LOW. Truncate-then-insert. Every customer gets exactly one action record.
run.py mart.crm_action_queue
dq_report.py — data quality HTML report
Auto-generate: null rates per column, outlier flags (IQR method on GMV + delivery delta), row counts per table, model metric summary. Output to reports/dq_report.html. Run this before and after each pipeline stage. Include in README as a screenshot.
